# Run locally

## Full stack (Docker)

The recommended way to run the platform is via the setup script, which launches all services in Docker.

<Snippet title="Start the full stack" value="bash scripts/setup.sh" type="shell">
{`$ bash scripts/setup.sh`}
</Snippet>

This starts the complete platform including the backend, database, MQTT broker, Wazuh SIEM, telemetry simulator, and ML pipeline. See **Installation** for the full breakdown of what the script does.

## Frontend dev server (optional)

If you're working on the Next.js UI (`www`) and want hot-reload outside Docker, you can run the dev server locally while the backend stays in Docker.

<Snippet title="Start frontend dev server" value="cd www && pnpm install && pnpm dev" type="shell">
{`$ cd www\n$ pnpm install\n$ pnpm dev`}
</Snippet>
